Week 13 – Lab 1: Windows Event Log Categories Case Study

1. Lab Overview 

The goal of this lab was to explore the main Windows Event Logs and understand what type of information each log contains. Windows records system activity inside logs, and these logs are very important when investigating security incidents.

In this lab, I opened Event Viewer inside a Windows 11 virtual machine and examined the three main log categories:

Application Logs

System Logs

Security Logs

I also practiced filtering logs and examining event details to understand what type of information analysts can collect during an investigation.

This lab helps build the foundation for later labs where I will detect failed logins, port scans, and other suspicious activity.

2. Environment

Platform: Oracle VirtualBox
Operating System: Windows 11 Virtual Machine
Tool Used: Windows Event Viewer

Event Viewer is a built-in Windows tool that allows analysts to view logs generated by the operating system and applications.

3. Data Sources

The following Windows logs were examined during the lab:

Application Log

Records events generated by installed applications and software.

Examples include:

Application crashes

Database errors

Software warnings

Program startup failures

System Log

Records events related to the Windows operating system and system services.

Examples include:

Driver issues

Network problems

Service failures

Hardware errors

Security Log

Records events related to authentication and security activity.

Examples include:

Successful logins

Failed login attempts

User privilege changes

Account activity

Security logs are especially important during incident response investigations.

5. Key Findings

During the investigation, several useful observations were made.

Application Log Findings

Warning events related to the Windows WebCache database

ESENT events indicating slow operations

These events usually relate to application performance or background processes.

System Log Findings

Time synchronization warning

Disk and storage related events

TPM related errors

System logs often contain warnings even when the system is working normally.

Security Log Findings

Multiple successful login events (Event ID 4624)

Privilege assignment events (Event ID 4672)

These events confirm that Windows logs authentication activity.

6. Why These Logs Matter in Security

These logs play a major role in security investigations.

For example:

Security logs can help detect:

brute force login attempts

unauthorized account access

privilege escalation

System logs can reveal:

malware affecting services

system crashes

network failures

Application logs can reveal:

exploited applications

software vulnerabilities

program crashes related to attacks

Together, these logs help analysts reconstruct what happened on a system.
